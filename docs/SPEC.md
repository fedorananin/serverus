# Serverus — техническое задание (v1)

Менеджер SSH/SFTP/FTP/S3-подключений для macOS. Заменяет связку
electerm + Cyberduck: терминал, двухпанельный файловый менеджер и туннели в
одном окне, с единым зашифрованным хранилищем и разблокировкой по Touch ID.

- **Статус:** личное использование → позже open source (MIT)
- **Платформа:** macOS (Apple Silicon), архитектура не должна мешать будущим Linux/Windows-сборкам
- **Язык UI и кода:** английский. Документация — русский/английский.

---

## 1. Стек

| Слой | Технология |
|---|---|
| Оболочка | Tauri 2 |
| Бэкенд | Rust (stable) |
| Фронтенд | Svelte 5 + TypeScript + Vite |
| Терминал | xterm.js |
| SSH/SFTP | `russh` + `russh-sftp` (чистый Rust, async); ключи/agent — `russh-keys` |
| FTP/FTPS | `suppaftp` (+ rustls для FTPS) |
| S3 | `aws-sdk-s3` (официальный SDK; кастомные endpoint'ы для DO Spaces, R2, B2, MinIO, …) |
| Крипто | `argon2`, `aes-gcm`, `rand` (только аудированные крейты, никакой своей криптографии) |
| Keychain / Touch ID | `security-framework` (SecAccessControl, biometryCurrentSet) |
| Слежение за файлами | `notify` (FSEvents) для авто-заливки при редактировании |

Обоснование: Rust-бэкенд держит все соединения и передачи вне UI-потока;
Svelte — минимум бойлерплейта; бинарник ~10–15 МБ.

## 2. Хранилище (vault)

### 2.1 Принцип

**Ровно один файл** `*.serverus` — в нём всё: дерево папок, соединения, секреты,
настройки приложения, известные host keys. Файл самодостаточен и переносим:
бэкап/синк = копирование одного файла. На новом устройстве расшифровывается
мастер-паролем.

### 2.2 Криптосхема (envelope)

```
Мастер-пароль ──Argon2id(salt, m=64MiB, t=3, p=4)──▶ KEK
KEK ──AES-256-GCM──▶ расшифровывает DEK (случайный 256-бит ключ данных)
DEK ──AES-256-GCM──▶ расшифровывает payload (весь vault)
```

Формат файла: открытый заголовок (magic `SRVS`, версия формата, параметры KDF,
salt, nonce, зашифрованный DEK) + единый зашифрованный payload (JSON).
Соль и параметры KDF — не секрет. Каждый акт шифрования — новый случайный nonce.

- Смена мастер-пароля = перешифровка только DEK (данные не трогаются).
- Запись атомарная: пишем во временный файл → `rename`. Перед записью — копия
  предыдущей версии в `<vault>.bak` (одна, перезаписываемая).
- Расположение по умолчанию: `~/Library/Application Support/Serverus/default.serverus`,
  в настройках можно выбрать любой путь (для iCloud/Dropbox-синка).

### 2.3 Touch ID

- После первой разблокировки мастер-паролем DEK кладётся в Keychain с
  `kSecAttrAccessibleWhenUnlockedThisDeviceOnly` + `biometryCurrentSet`.
- Дальше обычный запуск: Touch ID → DEK из Keychain → vault открыт. Мастер-пароль —
  всегда работающий fallback (кнопка «Use master password»).
- Изменение набора отпечатков в системе инвалидирует запись → просто просим
  мастер-пароль заново и перекладываем DEK.
- Мастер-пароль **нигде не хранится** — ни в Keychain, ни в памяти дольше, чем
  нужно для вывода KEK.

### 2.4 Блокировка

- Auto-lock: по таймауту неактивности (настройка, по умолчанию 15 мин; 0 = никогда)
  и при переходе Mac в сон. Блокировка = DEK и все расшифрованные секреты
  зачищаются из памяти (`zeroize`), активные соединения продолжают жить,
  но новые пароли из vault не достать до разблокировки.
- Открытые сессии при блокировке НЕ рвутся (пароль уже отдан серверу).

## 3. Модель данных (payload vault)

```jsonc
{
  "version": 1,
  "tree": [                     // папки с неограниченной вложенностью
    { "type": "folder", "id": "…", "name": "Clients", "badge": {"kind":"emoji","value":"💼"},
      "children": [ …folders and connections… ] }
  ],
  "connections": {
    "<id>": {
      "name": "prod-web-1",
      "badge": { "kind": "color", "value": "#e5484d" },   // или {"kind":"emoji","value":"🐘"}
      "protocol": "ssh" | "ftp" | "s3",  // ssh покрывает терминал+SFTP+SCP+туннели
      "host": "…", "port": 22,
      "auth": {
        "method": "password" | "key" | "agent",
        "username": "…",
        "password": "…",               // хранится только внутри vault
        "key_path": "~/.ssh/id_ed25519",  // или "key_inline": "-----BEGIN…"
        "key_passphrase": "…"
      },
      "jump_host": "<connection_id>|null",   // цепочка бастионов (ссылки на другие соединения)
      "ftp": { "tls": "explicit" | "none", "passive": true },
      "s3": {                          // только для protocol = "s3" (см. §4.4)
        "region": "fra1|null",         // регион подписи (не-AWS обычно принимает любой)
        "bucket": "my-space|null",     // null = в корне панели список всех бакетов
        "path_style": false,           // MinIO/self-hosted: https://endpoint/bucket/key
        "public_base_url": "https://cdn.example.com|null",  // база для «Copy public URL»
        "upload_acl": "private" | "public_read" | "ask"     // ACL загружаемых файлов
      },
      "remote_dir": "/var/www",        // стартовая удалённая папка (опц.)
      "local_dir": "~/Projects/…",     // стартовая локальная папка (опц.)
      "tunnels": [                     // сохранённые пробросы портов
        { "name": "mysql", "type": "local", "local_port": 3306,
          "remote_host": "127.0.0.1", "remote_port": 3306, "autostart": false }
      ],
      "notes": "…"                     // свободный текст
    }
  },
  "known_hosts": { "<host:port>": "<algo> <fingerprint>" },
  "settings": { …все настройки приложения, см. §8… }
}
```

Правила:

- Папка и соединение имеют бейдж: **эмодзи или цветной кружок** (палитра из
  ~10 предустановленных цветов + произвольный hex). Никаких картинок/иконок в v1.
- `jump_host` ссылается на другое сохранённое соединение → цепочки любой длины,
  конфиг бастиона не дублируется.
- Пароли и passphrase живут только внутри зашифрованного payload.

## 4. Протоколы и соединения

### 4.1 SSH (одно соединение — три роли)

Один конфиг SSH-соединения даёт:

1. **Терминал** — PTY-канал, xterm.js.
2. **Файлы** — SFTP-подсистема поверх той же SSH-сессии (это и есть «SCP-режим»;
   при недоступности SFTP-подсистемы — fallback на SCP-протокол для передачи).
3. **Туннели** — local port forwarding (`direct-tcpip`).

Всё мультиплексируется в **одной** TCP/SSH-сессии: открыл терминал → файловая
панель и туннель не требуют повторной авторизации.

- **Аутентификация:** пароль, ключ (ed25519/ECDSA/RSA, с passphrase и без),
  ssh-agent (`SSH_AUTH_SOCK`). Порядок попыток: agent → ключ → пароль
  (только заданные в конфиге методы). Ключ задаётся файлом (нативный
  file picker) или текстом (`key_inline` — хранится в vault и переезжает
  вместе с бэкапом файла).
- **Jump host (v1):** локаль → бастион → `direct-tcpip` → SSH-сессия к цели.
  Рекурсивно для цепочек.
- **Host key verification:** при первом подключении — диалог с fingerprint
  (SHA256) → принять и сохранить в vault / отклонить. При смене ключа —
  жёсткое предупреждение (красный экран, не «ОК по умолчанию»).
- **Keep-alive:** server-alive-interval 30 с; авто-reconnect с восстановлением
  удалённой папки (терминальная сессия честно объявляется потерянной).

### 4.2 Туннели (v1)

- Local forwarding: `localhost:<port>` → через SSH → `<remote_host>:<remote_port>`.
- Сохраняются в конфиге соединения, запускаются вручную из вкладки соединения
  (флаг `autostart` — вместе с подключением).
- Индикатор активных туннелей + счётчик трафика. Remote/dynamic (SOCKS) forwarding — v2.

### 4.3 FTP / FTPS

- Passive mode по умолчанию; explicit FTPS (AUTH TLS) поддерживается, установка
  «FTPS required» в конфиге запрещает откат на plaintext.
- Пул из N соединений на сервер для параллельных передач (см. §6).
- **Критично (боль electerm): рекурсивные операции над папками обязаны работать** —
  upload/download/delete папки любой вложенности. Это первый интеграционный тест проекта.

### 4.4 S3-совместимые хранилища

Поддерживаются все S3-совместимые провайдеры через кастомный endpoint:
AWS S3, DigitalOcean Spaces, Cloudflare R2, Backblaze B2, Wasabi, MinIO и т.п.
В форме подключения — пресеты провайдеров (заполняют endpoint-шаблон и
path-style), но в итоге это просто `host` + ключи.

- **Конфиг:** `host` = endpoint (HTTPS, порт из поля port, 443 по умолчанию),
  `username` = Access Key ID, `password` = Secret Access Key (живёт в vault,
  как все секреты). Регион — поле `s3.region` (для не-AWS обычно произвольный).
- **Бакеты:** поле `s3.bucket` опционально. Задано — панель открывается внутри
  бакета; пусто — корень панели показывает все бакеты аккаунта как папки
  (mkdir/rmdir на этом уровне = create/delete bucket).
- **Файловая модель:** S3 — плоское пространство ключей; «папки» — префиксы с
  разделителем `/` (ListObjectsV2 + CommonPrefixes). mkdir создаёт
  placeholder-объект `dir/`. Всё ходит через тот же trait `RemoteFs`; UI и
  очередь передач не знают, что это S3. Переименование = Copy + Delete
  (для папок — по всем объектам префикса). Докачка (resume) для S3-загрузок
  недоступна — retry перезапускает файл целиком (`RemoteFs::supports_resume`).
- **Загрузка:** стриминг через multipart upload (части по 8 МиБ; мелкие файлы
  — одним PutObject). Незавершённые multipart'ы прерываются (AbortMultipartUpload)
  при отмене/ошибке.
- **Права доступа (ключевая фича):** POSIX-прав нет — вместо chmod у объектов
  ACL public-read / private.
  - Колонка Mode в панели показывает public/private; статусы подгружаются
    **фоном** после листинга (GetObjectAcl — отдельный запрос на объект,
    батчами с ограниченной параллельностью) и кэшируются.
  - Контекстное меню: **Make public / Make private** — массово по выделению,
    для папок рекурсивно по всему префиксу.
  - **Режим загрузки** `s3.upload_acl`: `private` | `public_read` | `ask`
    (спросить перед загрузкой, один раз на батч). Переключалка — прямо в
    заголовке удалённой панели; выбор персистится в конфиге соединения.
  - Провайдеры без per-object ACL (Cloudflare R2, AWS-бакеты с
    Object Ownership = enforced) отклоняют эти запросы — показываем понятную
    ошибку, UI не прячем.
- **Copy public URL:** пункт контекстного меню. База URL — `s3.public_base_url`
  (CDN/кастомный домен), иначе строится из endpoint'а
  (virtual-host или path-style — по конфигу).
- Терминала и туннелей у S3-вкладок нет (как у FTP) — только Files.

## 5. Интерфейс

### 5.1 Общая структура

```
┌──────────┬──────────────────────────────────────────────┐
│ Sidebar  │  Tabs: [prod-web-1] [client-ftp] [+]         │
│          │ ┌──────────────────────────────────────────┐ │
│ дерево   │ │  Вкладка соединения:                     │ │
│ папок и  │ │  внутренние виды: Files | Terminal | Tunnels │
│ серверов │ │                                          │ │
│          │ ├──────────────────────────────────────────┤ │
│ 🔍 поиск │ │  Transfer queue (сворачиваемая панель)   │ │
└──────────┴──┴──────────────────────────────────────────┘
```

- **Sidebar:** дерево папок/соединений, drag&drop для переупорядочивания,
  контекстное меню (connect, edit, duplicate, delete, new folder), живой поиск
  по имени/хосту. Двойной клик — подключение в новой вкладке.
- **Вкладки:** каждая вкладка = одна сессия к серверу. Внутри — переключатель
  Files / Terminal / Tunnels (для FTP и S3 — только Files; Tunnels скрыта,
  пока у соединения нет настроенных туннелей). Можно открыть две вкладки
  на один сервер. Cmd+T / Cmd+W / Cmd+1..9.
  Переполнение (в стиле Safari): вкладки равномерно ужимаются до минимума
  (~96 px), дальше полоса скроллится по горизонтали (трекпад/колесо), края
  подсвечиваются градиентом-подсказкой, активная вкладка автоматически
  подъезжает в видимую зону; крестик закрытия — только на hover и на
  активной вкладке.
- **Разблокировка:** экран с Touch ID (авто-запрос при старте) и полем мастер-пароля.

### 5.2 Файловый режим — две панели

- **Слева локальные файлы, справа сервер.** Каждая панель: хлебные крошки/строка
  пути (редактируемая), список (имя, размер, дата, права), сортировка по колонкам,
  toggle скрытых файлов, быстрый фильтр по подстроке, virtual scrolling
  (10k+ файлов без лагов).
- **Выделение:** клик; Cmd+клик — toggle; Shift+клик — диапазон; Cmd+A — всё;
  рамка мышью (marquee) — с пустого места или с области строки правее имени
  файла (перетаскивание за само имя начинает drag&drop, как в Explorer).
- **Перенос:** drag&drop между панелями, drag&drop из Finder, кнопки ⇄, шорткаты
  (Cmd+стрелка в сторону панели). Копирование по умолчанию; перемещение — с Alt.
- **Операции:** new folder, new file, rename (F2/Enter), delete (с подтверждением,
  рекурсивно), refresh, chmod (см. 5.4), copy path, download/upload.
- Symlink'и отображаются со стрелкой и ведут по назначению.
- Server-to-server (обе панели удалённые) — v2.

### 5.3 Редактирование удалённых файлов

- Двойной клик (или «Edit») по файлу на сервере → скачивается во временную
  папку → открывается в редакторе (настройка: системный default / конкретное
  приложение, например VS Code).
- FSEvents следит за файлом: сохранил в редакторе → автоматическая заливка
  обратно + ненавязчивое уведомление «Uploaded ✓».
- Временные копии живут в изолированной папке, чистятся при закрытии вкладки/приложения.

### 5.4 Права доступа

Диалог chmod: сетка чекбоксов rwx × owner/group/others + синхронизированное
octal-поле (755 ⇄ чекбоксы). Для папок — опция «apply recursively»
(с выбором: files only / dirs only / both). Работает для SFTP и FTP (SITE CHMOD).
Для S3 вместо chmod — public/private ACL (см. §4.4).

### 5.5 Терминал

- xterm.js: 256 цветов + truecolor, scrollback 10 000 строк, копирование
  выделением, Cmd+V вставка (с подтверждением при многострочной вставке),
  настройка шрифта/размера, поиск по буферу (Cmd+F).
- Несколько терминалов на одну вкладку соединения (внутренние табы) — каналы
  в рамках одной SSH-сессии.
- Вывод стримится из Rust событиями с батчингом (~16 мс), чтобы `yes`/`cat bigfile`
  не вешал UI.

### 5.6 Визуальный стиль

- **Только тёмная тема** в v1. Минимализм: без градиентов, теней и декора;
  системный шрифт (SF Pro / -apple-system), моноширинный для терминала и колонок
  размера/прав.
- Контраст текста ≥ WCAG AA. Один акцентный цвет. Все цвета — CSS-переменные
  (светлая тема добавится в v2 без переписывания).
- Нативные ощущения: скруглённое окно Tauri, вибрация не нужна, никаких кастомных
  рамок окна.

## 6. Передачи файлов

### 6.1 Очередь

- Единая очередь передач внизу окна: файл/папка, прогресс, скорость, ETA,
  pause / resume / cancel (по элементу и глобально).
- Параллельность: до **N одновременных файлов на сервер** (настройка, по умолчанию 5),
  очереди разных серверов независимы.
- Конфликты: диалог Overwrite / Skip / Rename с «Apply to all». Настройка
  политики по умолчанию.
- Докачка больших файлов после обрыва: SFTP — по offset, FTP — REST.
- **Авто-retry:** упавшая передача автоматически перезапускается до 2 раз
  с бэкоффом (1 с, 2 с) и докачкой, прежде чем остаться в Error для ручного
  retry (ручной retry снова взводит авто-попытки). tar-стримы не
  авто-ретраятся — их retry падает обратно на пофайловый режим по решению
  пользователя.
- Сохранение mtime при копировании (опция, по умолчанию вкл).

### 6.2 Ускорение мелких файлов (ключевая фича)

Проблема: тысячи мелких файлов по SFTP/FTP идут медленно из-за round-trip'ов
на каждый файл.

1. **SSH + папка → tar-туннель.** Если на удалённой стороне есть `tar`
   (проверка `command -v tar` при подключении):
   - download: `tar -cf - -C <dir> .` → поток по SSH-каналу → распаковка локально;
   - upload: локальная упаковка → `tar -xf - -C <dir>` на сервере.
   Прогресс — по байтам потока против предварительно посчитанного размера.
   Один поток вместо тысяч round-trip'ов — скорость близка к каналу.
2. **Fallback (нет tar / FTP):** параллельная очередь из §6.1 + pipelining
   внутри SFTP (не ждать ack каждого блока).
3. Переключение автоматическое, в UI видно применённый режим («via tar stream»).
   Опция «force plain transfer» на случай проблем.

## 7. Архитектура кода

### 7.1 Rust (src-tauri)

```
src-tauri/src/
├── vault/          # формат файла, крипто (Argon2id/AES-GCM), атомарная запись, Keychain/TouchID
├── session/        # менеджер сессий: реестр активных подключений, reconnect, keep-alive
│   ├── ssh.rs      # russh: auth, jump chains, PTY, каналы
│   ├── sftp.rs     # файловые операции поверх SSH
│   ├── ftp.rs      # suppaftp: пул соединений, рекурсивные операции
│   ├── s3.rs       # aws-sdk-s3: префиксы-как-папки, multipart, ACL (§4.4)
│   └── tunnel.rs   # port forwarding
├── transfer/       # очередь, параллелизм, tar-акселерация, докачка, прогресс-события
├── watcher/        # remote edit: temp-файлы + FSEvents + авто-upload
└── commands.rs     # Tauri-команды (тонкий слой: parse → вызов модуля → ответ)
```

- Единый trait `RemoteFs` (list/get/put/mkdir/rename/delete/chmod/…) — SFTP,
  FTP и S3 реализуют его, UI и очередь передач не знают о протоколе.
  S3-специфика (ACL, public URL) — отдельные `s3_*`-команды поверх сессии,
  не расширение трейта.
- События в фронт: `transfer:progress`, `terminal:data`, `session:state`,
  `vault:locked` — через Tauri events.
- Секреты в памяти — `zeroize` при блокировке/выходе. Секреты никогда не
  попадают в логи и в сообщения ошибок.
- macOS-специфика (Keychain/TouchID) изолирована за trait'ом — будущие платформы
  получат свою реализацию quick-unlock.

### 7.2 Frontend (Svelte + TS)

```
src/
├── lib/
│   ├── api/         # типизированные обёртки над invoke() и события
│   ├── stores/      # vault, sessions, tabs, transfers, settings
│   └── components/  # Sidebar, FilePane, TransferQueue, Terminal, dialogs…
└── routes/          # Unlock screen → Main screen
```

- Строгий TypeScript, типы Tauri-команд синхронизированы с Rust
  (генерация через `specta`/`tauri-specta` — один источник правды).

### 7.3 Качество (сразу под будущий open source)

- Комментарии, идентификаторы, коммиты — английский. MIT LICENSE с первого коммита.
- `cargo clippy -D warnings`, `rustfmt`, `eslint`+`prettier` — в pre-commit.
- Тесты: unit на vault (roundtrip шифрования, смена пароля, повреждённый файл),
  интеграционные на файловые операции против **реальных локальных серверов**:
  непривилегированный OpenSSH `sshd` на случайном порту (есть на macOS и в CI
  из коробки), in-process FTP-сервер (`libunftp`) — **включая рекурсивную
  заливку папки по FTP** — и in-process S3-сервер (`s3s` + файловый бэкенд с
  in-memory ACL) для файловых операций, передач и public/private ACL. *(Изменение против исходного плана с
  docker-контейнерами: docker не требуется вовсе, тесты работают локально и в
  CI без внешних зависимостей; фиксируется той же правкой, что и реализация.)*
- CI (GitHub Actions): build + clippy + тесты — готов к моменту публикации.

## 11. Риски

| Риск | Митигация |
|---|---|
| Зрелость `russh` (редкие серверы/алгоритмы) | Fallback-план: `ssh2` (биндинги libssh2) за тем же trait'ом; интеграционные тесты против реальных серверов |
| Пропускная способность Tauri IPC для терминала | Батчинг вывода, при нехватке — бинарный канал |
| tar недоступен/другой на сервере | Автодетект + прозрачный fallback на обычную передачу |
| Повреждение vault при сбое записи | Атомарный rename + `.bak`-копия |
| Забытый мастер-пароль | Восстановления нет by design — предупреждаем при создании vault |
